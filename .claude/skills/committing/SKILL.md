# Committing in amplify-category-api

## Prerequisites

Before committing, ensure you're on a feature branch, not `main` or `upstream`.

```bash
git branch --show-current
```

If on `main` or `upstream`, create a branch first. See the `branching-strategy` skill for which base to use.

Branch naming: `<type>/<short-description>` (e.g., `feat/add-inference-profiles`, `fix/null-pointer-error`)

## Commit Message Format

This repo uses **commitlint** with `@commitlint/config-conventional` and `@commitlint/config-lerna-scopes`.

```
<type>(<scope>): <subject>

<body>

<footer>
```

The header is required. Scope should be a package name (Lerna scope) when applicable.

### Commit Types

| Type | Purpose |
|------|---------|
| `feat` | New feature |
| `fix` | Bug fix |
| `chore` | Maintenance, deps |
| `refactor` | Refactoring (no behavior change) |
| `perf` | Performance improvement |
| `docs` | Documentation only |
| `test` | Test additions or corrections |
| `ci` | CI configuration |
| `revert` | Revert a previous commit |

### Scopes

Scopes should match Lerna package names (e.g., `amplify-graphql-generation-transformer`, `amplify-graphql-api-construct`). Use the short form without the org prefix.

### Subject Line Rules

- Imperative, present tense: "Add feature" not "Added feature"
- Lowercase first letter (commitlint convention for this repo)
- No period at the end
- Maximum 100 characters for the header line

### Body Guidelines

- Explain **what** and **why**, not how
- Imperative mood, present tense
- Include motivation for the change

### Footer

Reference issues:
- `Fixes #1234` — closes the issue on merge
- `Refs #1234` — links without closing

### Examples

```
fix(amplify-graphql-generation-transformer): handle inference profile ARNs

The generation transformer hardcoded foundation-model/ in Bedrock ARNs,
which fails for inference profiles that use inference-profile/ prefix.
Detect the model type from the ID and use the correct ARN format.

Fixes #2989
Co-Authored-By: Claude <noreply@anthropic.com>
```

```
feat(amplify-graphql-conversation-transformer): add regional inference profile support

Support model IDs with regional prefixes (us., eu., ap.) in addition
to the existing global. prefix for cross-region inference profiles.

Refs #2989
```

## AI-Generated Changes

When changes were primarily generated by a coding agent, include in the footer:

```
Co-Authored-By: Claude <noreply@anthropic.com>
```

## No Git Hooks

This repo has **no pre-commit or pre-push hooks**. Validation happens in CI. You are responsible for:

1. Running `yarn prettier-write` on changed files before committing
2. Running `yarn test` in affected packages before pushing
3. Ensuring commit messages follow conventional commit format (commitlint runs in CI)

## Staging Recommendations

- Stage specific files, not `git add -A`
- Never commit `.env`, credentials, or secrets
- Run prettier on staged files before committing
